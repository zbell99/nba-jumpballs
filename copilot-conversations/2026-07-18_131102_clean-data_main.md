## Conversation Summary

### Goal
Identify and clean consecutive game plays in jump ball data by removing sparse rows where athlete IDs are incomplete.

### Key Decisions & Implementation

#### 1. Identifying Consecutive Plays
- Created `collect_dupes()` function to find all groups of consecutive game plays (2+ plays in a row with sequential `game_play_number` values)
- Returns summary list with `game_id`, `start_game_play_number`, `end_game_play_number`, and `length` for each group
- More efficient than the original approach limited to 2-4 consecutive plays

#### 2. Sparsity Comparison Logic
- Refined understanding: comparing **sets of athlete IDs** rather than just counting nulls
- Athlete IDs from `athlete_id_1` and `athlete_id_2` are interchangeable (can flip between rows)
- A row is considered sparser if its athlete ID set is a **strict subset** of another row's set
- Example:
  - Row 1: `{495, 614}` vs Row 2: `{572}` → Keep both (different sets, no subset relationship)
  - Row 1: `{495, 614}` vs Row 2: `{495}` → Remove Row 2 (strict subset)

#### 3. Implementation
- Created `clean_dupes_by_sparsity()` function to remove sparser rows from consecutive groups
- Uses set operations (`<` for strict subset) to determine which rows to remove
- Iterates through each consecutive group and compares all row pairs

### Code Changes
- Refactored `jumpball_explo.py` with two main functions:
  1. `collect_dupes(df)` - identifies consecutive play groups
  2. `clean_dupes_by_sparsity(df, consecutive_groups)` - removes sparse rows
- Updated `main()` to display total groups found and rows removed
